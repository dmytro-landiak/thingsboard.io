The **Trendz View Latest** widget supports ThingsBoard dashboard aliases as a data source.
When an alias value changes — for example, when a user selects a device/asset from a list — the Trendz view automatically re-filters its data to show only that entity.

**Example:** A dashboard has 10 Machine devices. One widget lists all machines; when the user clicks a row, a Trendz line chart updates to show the temperature of the selected machine.

## How alias filtering works

```plaintext
  ThingsBoard Dashboard
  ┌───────────────────────────────────────────┐
  │  Entity List widget                       │
  │  datasource: all_devices alias            │
  │  on row click → updates selected_device  ─┼──┐
  └───────────────────────────────────────────┘  │
                                                  │ alias value
                                                  ▼ (entity name)
  ┌───────────────────────────────────────────┐
  │  Trendz View Latest widget                │
  │  datasource: selected_device alias        │
  │  Filter Name: "Machine"                   │
  │                                           │
  │  Trendz injects alias name into filter    │
  │  → chart shows data for selected device   │
  └───────────────────────────────────────────┘
```

## Setup

### Step 1 — Configure ThingsBoard aliases and widgets

<Steps>
1. Create an **all_devices** alias that resolves all devices of type **Machine**.

2. Create a **selected_device** alias of type **Entity from dashboard state** — this alias holds a reference to the currently selected device.

3. Add an **Entity List** widget using **all_devices** as its datasource.

4. Configure an **On row click** action on the Entity List widget that saves the selected device to the **selected_device** alias.

5. Save the dashboard.
</Steps>

### Step 2 — Prepare the Trendz view

<Steps>
1. Open the view in Trendz.

2. Add a **Machine** field to the **Filter** section. This enables filtering by machine name.

3. Save the view and copy its link.
</Steps>

### Step 3 — Add the Trendz View Latest widget

<Steps>
1. Edit the dashboard and add the **Trendz View Latest** widget from the **Advanced analytics Bundle**.

2. Set **selected_device** as the widget's datasource and use **name** as the key from the alias.

3. Switch to the **Advanced** tab:
   - Paste the copied Trendz view URL into **View URL**.
   - Enter **Machine** in **Filter Name** — this must match exactly the business entity field name used in the Trendz view filter.

4. Save the dashboard.
</Steps>

Now every time **selected_device** updates, the entity name is injected into the Trendz view filter and the chart refreshes automatically.

<Aside type="note">
If the Trendz view has multiple filters configured, the system matches the correct one by name. When the alias is empty (no entity selected yet), the view shows data for all devices.
</Aside>

## Horizontal bars

By default bars are drawn vertically. To switch to a horizontal layout:

<Steps>

1. Open **View Settings**.

2. Enable **Horizontal Bar** mode.

</Steps>

<Aside type="note">
Horizontal mode is only available when **Y-axis** contains exactly one field. If multiple fields are on Y-axis or fields are present in the **Series** section, the chart will not display in horizontal mode.
</Aside>

## Stacked bars

When a bar chart has multiple series or Y-axis fields, each series is shown side by side by default. You can switch to a stacked layout to show contribution of each series to the total.

Three modes are available:

| Mode | Description |
|------|-------------|
| Stacking off | Bars for each series rendered side by side (default) |
| Stacking on | Bars stacked on top of each other, showing absolute totals |
| 100% stacked | Bars normalized to 100%, showing relative proportion per group |

Control the stack mode in **View Settings**. Stacked mode can be combined with horizontal mode.

## Group by telemetry value

Use this pattern when a single telemetry payload contains both a numeric value and a category string, and you want to aggregate the numeric value per category.

**Example payload** вҖ” a machine reports the product type, the number of units produced, and the energy consumed per cycle:

```json
{
  "ts": 1540892498884,
  "amount": 12,
  "product": "glass",
  "energy": 218
}
```

To build a chart showing energy consumed per product type, broken down by machine:

<Steps>

1. Add the **product** field to the **Series** section with aggregation **UNIQ** вҖ” this groups the telemetry by product value.

2. Add the **energy** field to the **Y-axis** section with aggregation **SUM** вҖ” this sums energy for each product group.

3. Add the **Machine** name field to the **X-axis** section вҖ” this creates one bar group per machine.

</Steps>

## Value labels

Show or hide numeric value labels on each bar:

<Steps>

1. Open **View Settings**.

2. Toggle the **Labels** checkbox.

</Steps>

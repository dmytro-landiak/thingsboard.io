Collections are folders that organize Trendz views and control which users can see or edit them. Every user starts with a root collection called **Your collections**, and can create any number of nested subcollections inside it.

## Collection types

| Type | Description |
|------|-------------|
| **Your collections** (root) | Default folder available to all users; cannot be deleted. All subcollections are created inside it. |
| **Subcollections** | Folders nested inside the root or inside other subcollections. Nesting depth is unlimited. |
| **Favourites** | A personal list of frequently used views. Click the star icon in the views table to add or remove a view from Favourites. |

## Permissions and visibility

Three permission levels apply to both collections and individual views:

| Level | Who has it | What they can do |
|-------|------------|-----------------|
| **Full** (read, write, run) | Owner (creator) only | View, interact with, and save changes |
| **Limited** (read, run) | Higher-rank users who are not the owner | View and interact; cannot save changes |
| **None** | Same-rank users who are not the owner | Cannot see the collection or view at all |

```plaintext
  Tenant
  ├── sees all customer/subcustomer views (limited)
  ├── cannot save changes to views they don't own
  │
  Customer A                 Customer B
  ├── sees own views (full)  ├── sees own views (full)
  └── ✗ cannot see           └── ✗ cannot see
       Customer B's views         Customer A's views
```

To see collections and views belonging to lower-rank users, click **Show subcustomers collections and views**.

## Managing collections and views

### Create a collection

Click **Add new** inside an existing collection. The new subcollection appears within the collection where you clicked.

### Sort collections

Collections sort by name by default. To sort by date, click **Sort by date** next to the root collection name. The sort order resets to name after a page reload.

### Rename a collection

Click **Change name** on any subcollection to rename it. The root collection cannot be renamed.

### Manage individual views

Open the three-dot menu on any row in the views table to rename, move, copy, or delete that view.

Select multiple views using the checkboxes in the table, then use the bulk action toolbar to move or delete them all at once.

### Delete a collection

A collection can only be deleted when it is empty — it must contain no views and no non-empty subcollections. Move or delete all contents first, then delete the collection.

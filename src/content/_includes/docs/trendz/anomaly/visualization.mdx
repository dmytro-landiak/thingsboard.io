Trendz provides three approaches for visualizing anomaly data, suited to different use cases:

| Approach | When to use |
|----------|-------------|
| **Anomaly View Report** | Plug-and-play visualization — no configuration required. |
| **Anomaly Fields** | Flexible custom views; no need to save anomalies to ThingsBoard. |
| **Business Entity Fields** | Fine-grained control when anomaly scores are saved to ThingsBoard. |

## Anomaly View Report

The Anomaly View Report is a pre-built, fully configured view for anomaly visualization. It provides the same
functionality as the <DocLink product={props.product} path="anomaly/overview#review-tab">Review tab</DocLink>
in the anomaly model details, but as a standalone view that can be embedded in a ThingsBoard dashboard widget like any other Trendz view.

**Available settings:**

| Setting | Description | Location |
|---------|-------------|----------|
| **Default Anomaly Selection** | Which anomaly is selected when the view opens: `last` or `max score index`. | *Settings → General → Default anomaly selection* |
| **Hide Anomalies List** | Removes the anomaly list widget from the view. | *Settings → View mode fields → Hide anomalies list* |
| **Hide Anomalies Chart** | Removes the anomaly chart widget from the view. | *Settings → View mode fields → Hide anomalies chart* |
| **Time Range** | Time range used for the view. | Upper centre of the view |
| **Filters** | Which items are shown in the view. | Upper left of the view |

## Anomaly Fields

Anomaly Fields are low-level building blocks for constructing custom anomaly visualizations inside any Trendz view —
timeseries, card, bar chart, and others. They query anomaly data directly from the model without requiring scores to
be saved to ThingsBoard. Multiple Anomaly Fields from different models can be added to the same view.

Use the filter panel to narrow results by score range or severity threshold.

**To add an anomaly field:**

1. Open any Trendz view.
2. Drag and drop an **Anomaly Field** onto the canvas.
3. Click the field to configure its properties.

**Field properties:**

| Property | Options | Description |
|----------|---------|-------------|
| **Anomaly** | `SCORE`, `SCORE_INDEX` | Which metric to visualize. |
| **Model** | (list of READY models) | The anomaly model to source data from. |
| **Aggregation** | AVG, MIN, MAX, SUM, COUNT, LATEST, UNIQ | How values are combined across time or entities (see guidance below). |
| **Label** | text | Field name shown in the chart legend. |
| **Chart Type** | `bar`, `line`, `area` | Visual representation style. |

**Aggregation guidance:**

`SCORE` measures anomaly **intensity** at a point in time — similar to a temperature reading:

| Aggregation | Suitability |
|-------------|-------------|
| `MAX` | ✓ Recommended — captures peak intensity |
| `LATEST` | ✓ Recommended — shows the most recent value |
| `COUNT` | ✓ Recommended — counts anomaly occurrences |
| `SUM`, `AVG`, `MIN`, `UNIQ` | ✗ Not recommended — lose meaningful signal |

`SCORE_INDEX` measures anomaly **cumulative impact** over time — similar to area under a curve:

| Aggregation | Suitability |
|-------------|-------------|
| `SUM` | ✓ Recommended — shows total anomaly footprint |
| `COUNT` | ✓ Recommended — counts anomaly occurrences |
| `UNIQ` | ✓ Recommended |
| `LATEST` | ✓ Recommended |
| `AVG`, `MIN` | ~ Acceptable but less sensitive |

## Anomaly business entity fields

When <DocLink product={props.product} path="anomaly/save-to-tb">Save to ThingsBoard</DocLink> is enabled for
the model, ThingsBoard stores anomaly scores as regular device telemetry. These can then be added to Trendz views as
**Business Entity Fields**.

<Aside type="note">
  Business Entity Fields are only available when the **Enable Saving Anomaly Data** toggle is active in the model's
  Refresh settings.
</Aside>

**To add a business entity field:**

1. Open any Trendz view.
2. Navigate to a business entity linked to the anomaly model.
3. Drag and drop one of the following fields:
   - `{Model Name} SCORE`
   - `{Model Name} SCORE INDEX`

## Choosing the right approach

| Goal | Use |
|------|-----|
| Quick visualization with no configuration | Anomaly View Report |
| Count anomalies without saving to ThingsBoard | Anomaly Fields with `COUNT` aggregation |
| Build a flexible custom view across devices | Anomaly Fields |
| Visualize saved scores with precise control | Business Entity Fields |

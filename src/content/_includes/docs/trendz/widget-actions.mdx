Trendz widgets embedded on ThingsBoard dashboards can trigger actions when a user interacts with them.
Actions let you connect Trendz views to the rest of the dashboard — for example, clicking a table row can select an entity, update the time window, or populate an alias used by other widgets.

## On row click — Table view

The **On row click** action is available only for the **Table** and **Anomaly table** view types.

The **Trendz Table** view supports an **On row click** action. When a user clicks a row, the action can save the entity to a dashboard state alias, open a new dashboard state, or run a custom script.

### Configure the action

<Steps>
1. Add the Trendz Table View widget to a ThingsBoard dashboard.

2. Open the widget in edit mode and switch to the **Actions** tab.

3. Click **Add action**.

4. In the **Action source** field, select **On row click**.

5. Complete the standard widget action configuration (action type, target state, etc.).
</Steps>

### Define which entity is selected on row click

Each row in a Trendz table can span data from multiple devices or assets. You need to tell Trendz which entity type represents "the selected row".

<Steps>
1. Open the view in Trendz and go to **View Settings**.

2. Expand the **View Mode fields** section.

3. Select the required device or asset type in the **Row click entity** dropdown.

4. Save the view.
</Steps>

## On row click — Anomaly table

If you are using the **Anomaly Table** on a dashboard and want the row click to both select an entity and update the dashboard time window:

<Steps>
1. In the Trendz chart settings, set the **Row click entity** (see step above).

   <ImageGallery images={[
     {
       src: '/src/assets/images/trendz/row-click-anomaly.png',
       alt: 'Row click entity setting in Trendz anomaly view',
       caption: 'Set the Row click entity in Trendz chart settings',
     },
   ]} />

2. On the ThingsBoard dashboard, open the Anomaly widget in edit mode, switch to the **Actions** tab, and add a new action.

3. Set **Action source** to **On row click** and **Action type** to **Custom action**.

4. Use the following code to update both the selected entity and the dashboard time window:

   ```js
   let stateId = widgetContext.stateController.getStateId();
   let stateParams = widgetContext.stateController.getStateParams();
   stateParams.selectedDevice = {
     entityId: entityId,
     entityName: entityName,
   };

   widgetContext.dashboard.onUpdateTimewindow(
     additionalParams.startTs,
     additionalParams.endTs
   );
   widgetContext.stateController.updateState(stateId, stateParams);
   ```

   Replace `selectedDevice` with the alias name used in your dashboard state.
</Steps>

## Date selected action

The **Calendar** and **Anomaly table** views support a **date-selected** action that updates the ThingsBoard dashboard time window when the user selects a date or anomaly time range. This action can be configured on the same widget alongside an **On row click** action.

<Steps>
1. Add the Trendz view widget to a dashboard.

2. Open the widget in edit mode and go to the **Actions** tab.

3. Click **Add action**, set **Action source** to **date-selected**, and set **Action type** to **Custom action**.

4. Use the following code — `additionalParams` provides `startTs` and `endTs` for the selected range:

   ```js
   widgetContext.dashboard.onUpdateTimewindow(
     additionalParams.startTs,
     additionalParams.endTs
   );
   ```
</Steps>

## On row click — Select multiple entities

A single row click can also populate an alias with a list of entities. This is useful when each row in the Trendz table aggregates data from multiple child devices and you want to pass all of them to another widget.

**Example:** A table shows buildings with aggregated energy consumption. Clicking a row sends all energy meters of that building to a dashboard alias.

```plaintext
  ┌──────────────────────────────────────────────┐
  │  Trendz Table row (Building A)               │
  │  Row click entity: EM energy meter           │
  └───────────────────────┬──────────────────────┘
                          │ additionalParams.entityIds[]
                          ▼
  ┌──────────────────────────────────────────────┐
  │  Dashboard alias: "Selected entities"        │
  │  Resolve as multiple entities: ON            │
  │  Filter type: Entity list                    │
  │  → updated with IDs from clicked row         │
  └──────────────────────────────────────────────┘
```

### Step 1 — Configure the Trendz view

<Steps>
1. In the Trendz view settings, set **Row click entity** to the entity type you want to send (e.g. **EM energy meter**).

   <ImageGallery images={[
     {
       src: '/src/assets/images/trendz/set-up-trendz-view.png',
       alt: 'Row click entity set to EM energy meter in Trendz view settings',
       caption: 'Set the Row click entity to the type you want to pass to the alias',
     },
     {

       alt: 'Trendz table showing buildings with energy meter counts',
       caption: 'Each row aggregates data from multiple child energy meters',
     },
   ]} />

2. Share the view to the ThingsBoard dashboard.
</Steps>

### Step 2 — Create the alias on the dashboard

<Steps>
1. Create a new alias and give it a name (e.g. **Selected entities** — you will reference this name in the action code).

2. Enable **Resolve as multiple entities**.

3. Set **Filter type** to **Entity list**.

4. Set **Entity type** to the type that will be selected via row click.

5. Optionally select default entities to show before any row is clicked.

   <ImageGallery images={[
     {
       src: '/src/assets/images/trendz/create-new-alias.png',
       alt: 'Dashboard alias configured as a multi-entity entity list',
       caption: 'Alias setup: Resolve as multiple entities + Entity list filter type',
     },
   ]} />
</Steps>

### Step 3 — Configure the custom action

<Steps>
1. In the Trendz widget on the dashboard, go to the **Actions** tab and add a new action.

2. Set **Action source** to **On row click** and **Action type** to **Custom action**.

   <ImageGallery images={[
     {
       src: '/src/assets/images/trendz/configure-action.png',
       alt: 'Custom action configuration with On row click source',
       caption: 'Action source: On row click; Action type: Custom action',
     },
   ]} />

3. Add the following code — it finds the alias by name, extracts the entity IDs from the clicked row, and updates the alias:

   ```js
   // Find the alias by name
   let entityAlias = Object.values(widgetContext.aliasController.entityAliases)
     .find(el => el.alias === 'Selected entities');

   // Update alias with entity IDs from the clicked row
   if (entityAlias && additionalParams.entityIds?.length) {
     entityAlias.filter.entityList = additionalParams.entityIds
       .map(entity => entity.entityId.id);
     widgetContext.updateAliases();
   }
   ```

   Replace `'Selected entities'` with the exact alias name you created in Step 2.
</Steps>

<Aside type="note">
`additionalParams.entityIds` is always populated on row click. The **Row click entity** setting determines which entity type's IDs are included — this matters when a table spans multiple asset or device profiles, since each profile contributes its own set of entities to the row.
</Aside>

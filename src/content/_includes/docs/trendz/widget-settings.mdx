Trendz widgets embedded on ThingsBoard dashboards support two settings that control how they refresh data and how they share filter state with other widgets on the same dashboard.

## Auto refresh

You can configure a Trendz widget to update its data automatically at a fixed interval.

<Steps>
1. Add the Trendz view widget to a ThingsBoard dashboard.

2. Open the widget in edit mode and switch to the **Advanced** tab.

3. Enable the **Update data in real time** checkbox.

4. Set the desired refresh interval in minutes.

5. Save the widget.
</Steps>

## Sharing filters between widgets

When a user changes the filter on one Trendz view, you can have other Trendz views on the same dashboard automatically update their filters to match. This is done by broadcasting a filter change event from a main view and listening for it on dependent views.

```plaintext
  ┌────────────────────────────────────────────────┐
  │  Main view                                     │
  │  Broadcast filter change event: ON             │
  │  Filter change event name: "myFilter"          │
  └──────────────────────┬─────────────────────────┘
                         │ filter change event
            ┌────────────┴────────────┐
            ▼                         ▼
  ┌──────────────────┐     ┌──────────────────┐
  │  Dependent view  │     │  Dependent view  │
  │  Listen: ON      │     │  Listen: ON      │
  │  Name: "myFilter"│     │  Name: "myFilter"│
  └──────────────────┘     └──────────────────┘
```

### Configure the main view

<Steps>
1. Open the main Trendz view widget in edit mode on the dashboard.

2. Switch to the **Appearance** tab.

3. Enable **Broadcast filter change event** — this publishes the current filter value whenever it changes.

4. Enter a name in **Filter change event name** (e.g. `myFilter`). All connected views must use this exact name.

5. Save the widget.
</Steps>

### Configure each dependent view

<Steps>
1. Open a dependent Trendz view widget in edit mode.

2. Switch to the **Appearance** tab.

3. Enable **Listen filter change event** — this view will now update its filters when it receives a broadcast.

4. Set **Filter change event name** to the same value used in the main view.

5. Save the widget. Repeat for every dependent view.
</Steps>

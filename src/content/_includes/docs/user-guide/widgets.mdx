A **widget** is a dashboard element that displays a specific type of data or provides specific functionality. Widgets enable visualization of device data, remote device control, alarm management, and display of custom HTML content.

Widgets are organized into **widget bundles**, which group related widgets by functionality or use case.

## Widget types

| Type | Description | Example |
|------|-------------|---------|
| **Time Series** | Display data across selected time windows — real-time rolling intervals or fixed historical periods. Designed for telemetry rather than attributes. | Line chart, Bar chart |
| **Latest Values** | Show the most recent values of selected attributes or telemetry keys. | Gauge, Value card |
| **Control** | Send RPC commands to your devices. | Knob, Switch |
| **Alarm** | Display and manage alarms and notifications. | Alarms table |
| **Static** | Present static content such as a floor plan or company information, including custom HTML. | Floor plan, HTML card |

See <DocLink product={props.product} path="reference/widgets/widget-library">Widgets Library</DocLink> for the complete list of widget bundles.

## Adding a widget to the dashboard

<Steps>
1. Open a dashboard and switch to **edit mode**.
2. Click **+ Add widget**.
3. Choose a **widget bundle &#8702; specific widget**.
4. **Configure data**
    - In **Datasources**, choose an <DocLink product={props.product} path="user-guide/aliases" bold={true}>entity alias</DocLink> (recommended) or select entities directly.
    - Add required **data keys** (<DocLink product={props.product} path='user-guide/digital-twins/time-series-data/' target="_blank">telemetry</DocLink>/<DocLink product={props.product} path='user-guide/digital-twins/attributes/' target="_blank">attributes</DocLink> fields as needed).
5. **Configure appearance and behavior**
    - Adjust widget settings (title, units, colors, thresholds, etc.).
    - (Optional) Configure <DocLink product={props.product} path="user-guide/actions/" bold={true}>actions</DocLink> (click behavior, navigation, RPC).
6. Click **Add** to add the widget to the dashboard.
7. Drag the widget to the desired position and resize it in the grid.
8. Click **Save** in the top right corner to apply changes and exit dashboard edit mode.
</Steps>

## Widget settings

Widget configuration includes setting up one or more [data sources](#data-source-types) (some widgets may have none), as well as configuring **appearance**, **widget card** parameters, **actions**, and **layout**.

### Data settings

#### Widget time window

The time window defines the time interval and aggregation/grouping rules applied to telemetry and alarm data.

A widget can use either its **own time window** configuration or inherit the **dashboard time window**.

See <DocLink product={props.product} path="user-guide/time-window" target="_blank">time window</DocLink> for complete details.

#### Data keys

A data key identifies the telemetry, attribute, or entity field a widget should use.

**Key sources:**
- **Attributes** — combined Client, Server, and Shared attributes of the selected entity.
- **Time series** — telemetry keys reported by devices or written by the Rule Engine/REST API.
- **Entity fields** — metadata fields depending on entity type (e.g., created time, name, type, label).

The available key sources depend on the widget type:
- **Time series widgets** — support time series keys with time window configuration.
- **Latest values widgets** — support time series, attributes, and entity fields.
- **Control and Static widgets** — do not require a data key.
- **Alarm widgets** — support all key types including alarm-specific fields.

To add a key, click the **Data keys** row and select from the dropdown. For keys not yet reported, type the name and choose its type (Attribute, Entity field, or Time series). The widget will start showing values once data becomes available.

**Delta calculation** — calculates the delta between the aggregated value for the current interval and a historical reference period. Available comparison periods:

- **Previous interval** — default; accounts for the current interval type.
- **Day ago** — subtracts 24 hours from the interval start and end.
- **Week ago** — shifts the interval back by 168 hours.
- **Month ago** — uses the same interval from the previous calendar month.
- **Year ago** — uses the equivalent interval from 365 days prior.
- **Custom interval** — specify a custom offset.

The delta result can be shown as:
- **Previous Value** — the aggregated value of the comparison interval.
- **Delta (absolute)** — the difference between the current and previous intervals (default).
- **Delta (percent)** — the relative difference as a percentage of the previous interval.

**Data post-processing function** — enables real-time transformation of incoming telemetry values before they are displayed. Use custom JavaScript to convert units, filter anomalies, or compute derived metrics. Available parameters:

- **time** — timestamp of the current value.
- **value** — the current incoming value.
- **prevValue** — the value returned by the previous function call.
- **timePrev** — timestamp of the previous value.
- **prevOrigValue** — the original previous value before post-processing.

The function must return a value (`number | string | boolean | null`). Returning `null` excludes the data point from the visualization.

#### Filters

Filters narrow which data a widget retrieves from its data sources. Build a filter by adding keys and defining conditions on those keys — multiple keys and conditions can be combined.

**Available key types:**
- **Entity field** — built-in metadata fields (e.g., name, type, label, createdTime).
- **Attribute** — any generic attribute key.
- **Client attribute** — attributes set by the device or edge client.
- **Server attribute** — attributes stored on the platform side.
- **Shared attribute** — configuration attributes delivered from the server to the device.
- **Time series** — telemetry keys representing values over time.

**Operations:** equal, not equal, greater than, less than, greater or equal, less or equal. Comparison values can be fixed literals or dynamic sources that pull values at runtime.

#### Alarm filter

Alarm widgets support filtering by status, severity, and type. You can choose any combination of alarm statuses and severities, define specific alarm types, and enable **Search of propagated alarms** to include alarms propagated from child entities.

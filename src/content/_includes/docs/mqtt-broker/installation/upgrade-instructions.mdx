<ConditionalHeading level={2} id="notice-about-bitnami-images" exclude="mqtt-broker-pe">Notice about Bitnami images (v2.2.0)</ConditionalHeading>

<p>TBMQ deployments prior to v2.3.0 use <strong>Bitnami images</strong> for third-party components such as PostgreSQL, Redis, and Kafka.
The Bitnami team <a href="https://github.com/bitnami/charts/issues/35164" target="_blank" rel="noopener noreferrer">has announced important upcoming changes</a> to their catalog that affect these images.</p>

<ul>
  <li>Starting <strong>August 28, 2025</strong>, the public <code>docker.io/bitnami</code> registry will only provide a small set of "latest" hardened images for development use.</li>
  <li>All versioned images will be moved to a temporary legacy registry: <code>docker.io/bitnamilegacy</code>. These images will remain available but will not receive updates or security patches.</li>
  <li>The long-term supported solution is Bitnami's new Secure Images (subscription-based).</li>
</ul>

<p><strong>Recommended:</strong> Complete the upgrade to v2.2.0 first (upgrades must be sequential), then continue to v2.3.0, which replaces all Bitnami images with official open-source alternatives. See <a href="#third-party-component-updates-in-v230">third-party component updates in v2.3.0</a> for details.</p>

<p>If you are staying on v2.2.0 for now, update your image references from <code>docker.io/bitnami/...</code> to <code>docker.io/bitnamilegacy/...</code> to avoid disruption after August 28, 2025.</p>

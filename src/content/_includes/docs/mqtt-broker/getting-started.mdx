## Introduction

This guide walks you through the basics of <a href="/products/mqtt-broker/" target="_blank" rel="noopener noreferrer">TBMQ</a>:

- Connecting MQTT clients to the broker.
- Publishing messages and subscribing to topics.
- Configuring client authentication and authorization.

For an overview of how the broker is built, see <DocLink product={props.product} path="architecture">Architecture</DocLink>.

## Try live demo

The fastest way to get started with TBMQ is to use the free public broker at <a href="https://demo.tbmq.io" target="_blank" rel="noopener noreferrer">demo.tbmq.io</a>.
The sandbox lets you explore TBMQ features without installing anything.

**MQTT connection details:**

| Parameter | Value           |
|-----------|-----------------|
| Host      | demo.tbmq.io    |
| TCP port  | 1883            |
| TLS port  | 8883            |
| Username  | demo            |
| Password  | *(leave empty)* |

Use these credentials to start publishing and subscribing right away.
To explore sessions, subscriptions, and other features in the TBMQ UI, <a href="https://demo.tbmq.io/signup" target="_blank" rel="noopener noreferrer">sign up</a> for a free account.

<Aside type="note">
  The demo instance grants read-only access to the UI. You can observe sessions, subscriptions, and broker statistics, but you cannot modify the configuration.
  To run TBMQ with full administrative access, follow the [Install TBMQ locally](#install-tbmq-locally) section.
</Aside>

## Configure client authentication and authorization

<Aside type="note">
  If you are using the <a href="https://demo.tbmq.io" target="_blank" rel="noopener noreferrer">demo instance</a>, skip this section. The demo credentials are pre-configured and ready to use.
</Aside>

The bundled Docker setup ships with <DocLink product={props.product} path="security/authentication/basic">Basic authentication</DocLink> enabled, so you only need to add a set of client credentials for your MQTT client to use:

- Go to **Authentication** → **Credentials** and click **Add** (+).
- Enter a credentials name, for example, "Getting Started Credentials".
- Enter `username` and `password` values of your choice — for example, `username` and `password`.
- Click **Add**.

By default, the authorization rules attached to new credentials allow publishing to and subscribing to any topic.

## Publishing and subscribing to topics

To watch messages flow through the broker, open one client that subscribes to a topic and another that publishes to it.
The examples below use the <a href="https://mosquitto.org/" target="_blank" rel="noopener noreferrer">Mosquitto</a> CLI clients.
For all available options, see the <a href="https://mosquitto.org/man/mosquitto_pub-1.html" target="_blank" rel="noopener noreferrer">mosquitto_pub</a> and
<a href="https://mosquitto.org/man/mosquitto_sub-1.html" target="_blank" rel="noopener noreferrer">mosquitto_sub</a> documentation.

{/* DIAGRAM-PLACEHOLDER: pub/sub flow — Publisher → TBMQ broker → Subscriber, with the broker's topic-matching step in the middle */}

```text
   ┌──────────────┐     PUBLISH      ┌─────────────┐    PUBLISH     ┌──────────────┐
   │  Publisher   │ ───────────────▶ │    TBMQ     │ ─────────────▶ │  Subscriber  │
   │ mosquitto_pub│  sensors/temp    │    broker   │  sensors/temp  │ mosquitto_sub│
   └──────────────┘   m=32, QoS 1    └─────────────┘   m=32, QoS 1  └──────────────┘
                                            │
                                            └── matches subscription "sensors/temperature"
```

<Tabs>
<TabItem label="Demo Instance">

##### Subscribe to a topic

To subscribe to the **sensors/temperature** topic and start receiving messages, run:

```bash
mosquitto_sub -d -h demo.tbmq.io -p 1883 -t sensors/temperature -q 1 -u demo
```

After the client connects, you can view its session on the **Sessions** page.

If you signed up for a demo account, log in to <a href="https://demo.tbmq.io" target="_blank" rel="noopener noreferrer">demo.tbmq.io</a> and open the **Sessions** page to see all active sessions, including the new one.

##### Publish a message

To publish a message to the **sensors/temperature** topic, run:

```bash
mosquitto_pub -d -h demo.tbmq.io -p 1883 -t sensors/temperature -m 32 -q 1 -u demo
```

</TabItem>
<TabItem label="Local Deployment">

##### Subscribe to a topic

To subscribe to the **sensors/temperature** topic and start receiving messages, run:

```bash
mosquitto_sub -d -h localhost -p 1883 -t sensors/temperature -q 1 -u username -P password
```

<Aside type="note">
  Replace `username` and `password` with the credentials you created in the [Configure client authentication and authorization](#configure-client-authentication-and-authorization) section.
</Aside>

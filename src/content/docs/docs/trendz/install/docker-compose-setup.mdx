Trendz Analytics cluster deployment is built into the ThingsBoard PE Docker Compose setup. There is no separate cluster configuration for Trendz — it runs as a service within the same Docker Compose project as ThingsBoard PE.

Follow the full cluster setup instructions in <DocLink product={Products.PE} path="installation/docker-compose-setup">Cluster Setup using Docker Compose</DocLink>, then enable Trendz as described in the <DocLink product={Products.PE} path="installation/docker-compose-setup#step-8-enable-trendz-analytics-optional">Enable Trendz Analytics</DocLink> step.

## What Trendz gets in cluster mode

- Trendz runs as a dedicated container (`trendz`) inside the ThingsBoard PE compose project.
- It connects to the shared PostgreSQL instance managed by the cluster setup.
- The Python Executor service (`trendz-python-executor`) is deployed alongside Trendz automatically.
- HAProxy routes external traffic to Trendz on port `8888`.

## Upgrading Trendz

Trendz has its own release cycle and must be upgraded separately from ThingsBoard PE. See the <DocLink product={Products.PE} path="installation/docker-compose-setup#upgrading-trendz">Upgrading Trendz</DocLink> section in the cluster setup guide for the exact steps.

<Aside type="note">
You can upgrade Trendz from any version directly to the latest without intermediate steps.
</Aside>

This guide explains how to install **Trendz Analytics** on **Windows 10 or Windows 11** (32-bit/64-bit).

<Aside type="caution" title="Docker is recommended">
Certain Trendz features — **predictions, metric explorations, and Python calculation fields** — require the Trendz Python Executor, which runs only via Docker. Even on Windows, Docker is needed to fully utilize all Trendz capabilities.

Consider using the <DocLink product={Products.TRENDZ} path="install/docker-windows">Docker (Windows)</DocLink> installation guide instead. Proceed here only if you have a specific reason to use a Windows package setup.
</Aside>

## Installation steps

### Step 1. Install Java 17 (OpenJDK)

1. Visit the <a href="https://adoptium.net" target="_blank">Adoptium download page</a>. Go to **Other platforms and versions**, select **Operating System: Windows**, **Architecture: x64**, **Version: 17 - LTS**, and download the **JDK .msi** package.
2. Run the downloaded MSI and follow the installer. Make sure **Add to PATH** and **Set JAVA_HOME variable** are both set to **Will be installed on local hard drive**.
3. Visit the <a href="https://jdbc.postgresql.org/download/" target="_blank">PostgreSQL JDBC download page</a> and download the latest driver.
4. Create the folder `C:\Program Files\JDBC` and copy the downloaded `.jar` file there.
5. Add a global `CLASSPATH` variable — run PowerShell as Administrator and execute (replace the filename to match your downloaded version):

```powershell
[System.Environment]::SetEnvironmentVariable("CLASSPATH", '.;"C:\Program Files\JDBC\postgresql-42.2.18.jar"', [System.EnvironmentVariableTarget]::Machine)
```

Verify the installation from Command Prompt:

```text
java -version
```

Expected output:

```text
C:\Users\User>java -version
openjdk version "17.x.xx"
OpenJDK Runtime Environment Temurin-17.x.xx (...)
OpenJDK 64-Bit Server VM Temurin-17.x.xx (...)
```

### Step 3. Configure database

Trendz uses PostgreSQL. You can install it on the same server or use a managed PostgreSQL service.

**Install PostgreSQL:**

Download the PostgreSQL installer (version 12.17 or newer) from the <a href="https://www.enterprisedb.com/downloads/postgres-postgresql-downloads#windows" target="_blank">EnterpriseDB download page</a> and follow the installation instructions.

During installation you will be prompted to set a password for the `postgres` superuser — remember this password, it will be used later.

**Create the Trendz database:**

Once installed, open **pgAdmin**, log in as the `postgres` superuser, and create a new database named `trendz` with owner `postgres`.

**Configure the database connection:**

Open Notepad (or any text editor) **as Administrator** (right-click → Run as administrator). Open the following file — make sure to select **All Files** in the file type dropdown, encoding is UTF-8:

```text
C:\Program Files (x86)\trendz\conf\trendz.yml
```

Locate the `datasource` block and update the connection values. Replace `postgres` with your actual postgres user password:

```yml
datasource:
    driverClassName: "${SPRING_DRIVER_CLASS_NAME:org.postgresql.Driver}"
    url: "${SPRING_DATASOURCE_URL:jdbc:postgresql://localhost:5432/trendz}"
    username: "${SPRING_DATASOURCE_USERNAME:postgres}"
    password: "${SPRING_DATASOURCE_PASSWORD:postgres}"
    hikari:
      maximumPoolSize: "${SPRING_DATASOURCE_MAXIMUM_POOL_SIZE:5}"
```

### Step 4. Run the installation script

Open **Command Prompt as Administrator** and change directory to your Trendz installation folder:

```text
cd "C:\Program Files (x86)\trendz"
```

Run `install.bat` to register Trendz as a Windows service (it will start automatically on system startup). `uninstall.bat` removes it from Windows services.

```text
install.bat
```

Expected output:

```text
C:\Program Files (x86)\trendz>install.bat
Detecting Java version installed.
CurrentVersion 11
Java 11 found!
Installing Trendz Analytics...
...
Trendz Analytics installed successfully!
```

### Step 5. Start Trendz service

Open **Command Prompt as Administrator** and run:

```text
net start trendz
```

Expected output:

```text
The Trendz Analytics service is starting.
The Trendz Analytics service was started successfully.
```

To restart the service:

```text
net stop trendz
net start trendz
```

## Troubleshooting

Log files are located in `C:\Program Files (x86)\trendz\logs`.

Check `trendz.log` for a successful startup line:

```text
YYYY-MM-DD HH:mm:ss,sss [main] INFO  o.t.t.TrendzApplication - Started TrendzApplication in x.xxx seconds (JVM running for x.xxx)
```
